List three security risks associated with using AI as a development assistant.\Risks include sensitive data leaks, the AI suggesting vulnerable code, and creating a false sense of security.$
Explain the principle of "Garbage In, Garbage Out" in AI training.\If an AI model is trained on poor-quality or insecure code from public repositories, it will likely generate insecure code for its users.$
What is "Secure Prompt Engineering"?\It involves including explicit security requirements and secure patterns in prompts to guide the AI toward generating safer outputs.$
Define "Human-in-the-loop validation" for AI code.\It is the requirement that a human developer must always review and validate code generated by AI before it is integrated into a system.$
Contrast "Prescriptive" and "Descriptive" SDLC models.\Prescriptive models (like Microsoft SDL) tell you what you should do; Descriptive models (like BSIMM) observe what organizations actually do.$
List four of the "7 Touchpoints" for software security.\Examples include Code Review, Risk Analysis, Penetration Testing, and Security Requirements.$
Explain the goal of the BSIMM model.\BSIMM uses real-world data from many companies to describe and measure security activities, allowing firms to compare their progress against the industry.$
What is the difference between an SDLC and a Security Standard?\SDLC is a process/methodology (how to build); Security Standards (like ISO 27001) are requirements/benchmarks (what to build).$
Define "Responsible Disclosure" and its primary benefit.\It is the practice of notifying a vendor about a vulnerability before informing the public, allowing the vendor time to create a patch.$
Explain the purpose of a "Bug Bounty" program.\It offers financial rewards to external researchers for finding and reporting vulnerabilities, incentivizing ethical hacking over malicious exploitation.$
