What is Information Gathering in web security?\Information Gathering is the reconnaissance phase where an attacker maps a target's architecture, finds hidden files, and identifies server versions to find potential entry points.$
What is SQLi?\SQLi stands for SQL Injection; it is a vulnerability where an application improperly handles user input, allowing an attacker to insert malicious SQL fragments that the database executes as commands.$ How is SQLi exploited for authentication bypass?\Attackers inject a "tautology" like ' OR 1=1; -- into a login field, which tricks the database into returning a "true" result and granting access without a valid password.$
How is SQLi mitigated?\The primary defense is using prepared statements with bind variables, which ensures the database treats user input strictly as data and never as executable code.$ What is Blind SQLi?\Blind SQLi is a type of SQL injection where the database does not show data directly; the attacker instead learns information by asking "True/False" questions and observing boolean changes in the page or inducing time delays.$
What is XPath injection?\XPath injection is an attack targeting applications that use user input to construct XPath queries for XML data, allowing an attacker to bypass security filters or explore the XML structure.$ What is LDAP Injection?\LDAP stands for Lightweight Directory Access Protocol; this injection occurs when an attacker puts special characters into a search for user info to bypass authentication.$
What is XML Injection?\XML stands for eXtensible Markup Language; this attack involves injecting malicious tags into a data stream to alter the application's logic.$ What is SSI Injection?\SSI stands for Server-Side Includes; this targets web servers that evaluate SSI directives, allowing an attacker to execute arbitrary shell commands on the server.$
What is IMAP or SMTP Injection?\IMAP and SMTP are email protocols; this attack targets mail server commands via web forms to send unauthorized emails or access restricted mailboxes.$ What is Code Injection?\Code Injection is a critical vulnerability where an attacker injects and executes arbitrary programming code (like Python or PHP) directly into the server's runtime environment.$
Why is Session Management necessary in web applications?\Because HTTP is a "stateless" protocol, meaning the server forgets who a user is after every request. Session management allows the server to track a specific authenticated user across multiple requests.$ What is a Session ID?\A Session ID (or Token) is a unique, random string generated by the server and stored in the user's browser (usually as a cookie) to identify their active session.$
What is Session Token Theft via network sniffing?\An attack where an adversary intercepts unencrypted traffic to steal a user's Session ID, allowing them to hijack the active session.$ What is a Session Token Prediction attack?\An attack where a server generates Session IDs using a non-random pattern, such as a counter, allowing an attacker to guess a valid ID of another active user.$
What is Session Fixation?\An attack where the adversary provides a victim with a known Session ID and waits for them to log in. The server then elevates that ID to an authenticated state without changing it, giving the attacker access.$ How is Session Fixation mitigated?\Web applications must always invalidate the existing Session ID and issue a brand-new, random ID immediately upon a successful login.$
