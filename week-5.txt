What is the primary function of a CAPTCHA?\CAPTCHA stands for Completely Automated Public Turing test to tell Computers and Humans Apart; it is used to ensure a user is a real human and not an automated bot to prevent abuse.$
What is XSS?\XSS stands for Cross-Site Scripting; it occurs when an application includes untrusted data in a web page without proper validation, allowing malicious scripts to execute in a victim's browser.$ How is XSS typically exploited?\Attackers use XSS to hijack sessions by stealing cookies, redirecting users to malicious sites, or performing unauthorized actions as the logged-in user.$
What is the difference between Reflected and Stored XSS?\XSS stands for Cross-Site Scripting. Reflected is a "one-time" attack delivered via a URL parameter. Stored is "persistent" because the script is saved on the server database and hits every user who visits.$ What is DOM-based XSS?\DOM stands for Document Object Model; this is an XSS attack where the malicious script runs entirely in the user's browser by changing the page structure locally without the server's involvement.$
How is XSS mitigated?\XSS stands for Cross-Site Scripting. Mitigation involves context-aware output encoding (converting characters like < into text entities) and implementing a strong Content Security Policy (CSP).$ What is CSRF?\CSRF stands for Cross-Site Request Forgery; it is an attack that tricks a logged-in user's browser into sending an unauthorized request (like moving money) to a web application where they are already authenticated.$
How is CSRF mitigated?\CSRF stands for Cross-Site Request Forgery. The standard defense is the use of anti-CSRF tokens, which are unique, secret values generated by the server and included in every request to verify it is legitimate.$ What is SSRF?\SSRF stands for Server-Side Request Forgery; it occurs when an attacker tricks a web server into making requests to internal resources (like an admin panel) that the server can reach but the attacker cannot.$
What is XXE?\XXE stands for XML External Entities; it is a vulnerability in XML parsers that allows an attacker to define an external reference that tricks the server into disclosing local files.$ What is a Billion Laughs attack?\It is a type of XML Denial of Service (DoS) attack targeting parsers where nested entities expand exponentially, consuming all available memory and crashing the system.$
What is Clickjacking?\Also known as UI Redressing, it involves using transparent layers or iframes to trick a user into clicking a hidden button while they think they are clicking a safe, visible page.$ How is Clickjacking mitigated?\By using the X-Frame-Options or Content-Security-Policy: frame-ancestors headers to tell the browser not to allow the website to be loaded inside an iframe on another domain.$
What is Insecure Deserialization?\A vulnerability where an application takes untrusted "serialized" data (data in a byte stream) and converts it back into an object without validation, often leading to Remote Code Execution (RCE).$ What is IDOR?\IDOR stands for Insecure Direct Object Reference; it occurs when an application provides direct access to objects based on user input (like a database ID in the URL) without checking if the user is authorized for that specific object.$
What is the difference between LFI and RFI?\LFI stands for Local File Inclusion, which tricks the server into showing its own internal files. RFI stands for Remote File Inclusion, which tricks the server into fetching and executing a script from an external, attacker-controlled site.$ What is Command Injection?\A vulnerability where an attacker can execute arbitrary operating system commands on the server by exploiting unsanitized input passed to a system shell.$
What is a major limitation of automated vulnerability scanners?\Scanners are often ineffective at identifying complex logical flaws, multi-step authorization issues, or vulnerabilities that require an understanding of the application's specific business logic.$
