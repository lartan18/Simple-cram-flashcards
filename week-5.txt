What is the primary goal of Information Gathering in web security?\Information Gathering is the reconnaissance phase where an attacker maps the application's structure, fingerprints the web server, and identifies entry points to improve attack efficiency and find "low hanging fruit" vulnerabilities.$
What is SQLi?\SQLi stands for SQL Injection; it is a vulnerability where an application improperly handles user input, allowing malicious SQL fragments to be inserted and executed as commands by the backend database.$ How is SQLi exploited for authentication bypass?\An attacker injects a "tautology" (a statement that is always true) like ' OR 1=1; -- into a login field, which tricks the database into returning a "true" result and granting access without a valid password.$
How is SQLi mitigated?\The primary defense is using prepared statements with bind variables, which ensures the database treats user input strictly as data and never as executable code logic.$ What is Blind SQLi?\A type of SQL injection where the database does not return data directly to the screen; the attacker instead learns information by asking "True/False" questions and observing boolean changes in the page response or inducing time delays using functions like sleep().$
What is XPath injection?\An attack targeting applications that use user input to construct XPath queries (used for XML data), allowing an attacker to bypass security filters or explore the internal XML structure.$ What is LDAP Injection?\LDAP stands for Lightweight Directory Access Protocol; this injection occurs when an attacker inserts special characters into a search query to bypass authentication or view restricted directory data.$
What is XML Injection?\A vulnerability where an attacker injects malicious tags into an XML data stream to alter the application's logic or exploit how the XML parser processes data.$ What is SSI Injection?\SSI stands for Server-Side Includes; this targets web servers that evaluate SSI directives, allowing an attacker to execute arbitrary shell commands on the server by inserting them into a web page.$
What is IMAP or SMTP Injection?\IMAP and SMTP are email protocols; this attack targets mail server commands via web forms, allowing an attacker to send unauthorized emails or access restricted mailboxes.$ What is Code Injection?\Code Injection is a critical vulnerability where an attacker injects and executes arbitrary programming code (like Python or PHP) directly into the server's runtime environment.$
Why is Session Management necessary in web applications?\Because HTTP is a "stateless" protocol, meaning the server forgets who a principal is after every request; session management allows the server to track a specific authenticated user across multiple requests.$ What is a Session ID?\A Session ID (or Token) is a unique, random string generated by the server and stored in the user's browser (usually as a cookie) to identify their active authenticated session.$
What is Session Token Theft via network sniffing?\An attack where an adversary intercepts unencrypted HTTP traffic to steal a user's Session ID, allowing them to impersonate the victim and hijack the active session.$ What is a Session Token Prediction attack?\An attack where a server generates Session IDs using a non-random pattern (such as a counter), allowing an attacker to guess a valid ID belonging to another active user.$
What is Session Fixation?\An attack where an adversary provides a victim with a known Session ID (e.g., via a link) and waits for them to log in; the server then "elevates" that specific ID to an authenticated state without changing it, giving the attacker access.$ How is Session Fixation mitigated?\Web applications must always invalidate the existing Session ID and issue a brand-new, random ID immediately upon a successful login.$
