What is XSS?\XSS stands for Cross-Site Scripting. It occurs when an application includes untrusted data in a web page without proper validation or encoding, allowing malicious scripts to execute in a victim's browser. $ How is XSS typically exploited?\Attackers use XSS to hijack active sessions by stealing session cookies, redirecting users to malicious websites, or performing unauthorized actions as the logged-in user. $ What is the difference between Reflected and Stored XSS?\Reflected XSS is a "non-persistent" attack where the script is delivered via a URL parameter and executed only for the person who clicks the link. Stored XSS is "persistent" because the script is saved on the server (e.g., in a database) and executes for every user who visits the affected page. $ What is DOM-based XSS?\A type of XSS where the entire vulnerability exists in the client-side code. The malicious script is executed when the browser's JavaScript processes data from the Document Object Model (DOM) without necessarily involving the server. $ How is XSS mitigated?\Mitigation involves "context-aware output encoding" (converting characters like < to &lt; so the browser treats them as text, not code) and implementing a strong Content Security Policy (CSP). $ What is CSRF?\CSRF stands for Cross-Site Request Forgery. It is an attack that tricks a logged-in user's browser into sending an unauthorized, hidden request (like a "transfer money" command) to a vulnerable web application where the user is currently authenticated. $ How is CSRF mitigated?\The standard defense is the use of anti-CSRF tokens, which are unique, secret values generated by the server and included in every request. The server rejects any request that does not contain the correct token. $ What is SSRF?\SSRF stands for Server-Side Request Forgery. It occurs when an attacker tricks a web server into making requests to internal or external resources (like a private admin panel or a password file) that the server has access to, but the attacker does not. $ How is SSRF exploited?\Attackers often use SSRF to scan internal networks, access metadata services (like AWS/Azure configuration files), or read local system files using the file:/// protocol. $ What is XXE?\XXE stands for XML External Entities. It is a vulnerability in XML parsers that allows an attacker to define a reference to an external file. When the server parses the XML, it is tricked into disclosing local files or interacting with internal systems. $ What is a Billion Laughs attack?\A type of Denial of Service (DoS) attack targeting XML parsers where nested entities expand exponentially (e.g., one entity becomes ten, which each become ten more), eventually consuming all available server memory and crashing the system. $ What is Clickjacking?\Also known as "UI Redressing," this involves using transparent layers or iframes to trick a user into clicking a hidden button on a malicious site while they believe they are clicking on a safe, visible page. $ How is Clickjacking mitigated?\By using the X-Frame-Options or Content-Security-Policy: frame-ancestors headers to tell the browser not to allow the website to be loaded inside an iframe on another domain. $ What is Insecure Deserialization?\A vulnerability where an application takes untrusted "serialized" data (data converted into a byte stream for storage or transmission) and converts it back into an object without validation, often leading to Remote Code Execution (RCE). $ What is IDOR?\IDOR stands for Insecure Direct Object Reference. It occurs when an application provides direct access to objects (like a file or database record) based on user-supplied input (like changing a user ID in a URL) without checking if the user is authorized to see that specific object. $ What is the difference between LFI and RFI?\LFI (Local File Inclusion) tricks the server into displaying or running files already present on the local server disk. RFI (Remote File Inclusion) tricks the server into fetching and executing a malicious script hosted on a remote, attacker-controlled server. $ What is Command Injection?\A vulnerability where an attacker can execute arbitrary operating system commands (like rm -rf /) on the server by exploiting unsanitized input that is passed to a system shell. $ What is a major limitation of automated vulnerability scanners?\Scanners are often ineffective at identifying complex logical flaws, multi-step authorization issues, or vulnerabilities that require an understanding of the specific business logic of an application.
