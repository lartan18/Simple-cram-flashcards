What is a substitution cipher and how is it broken?\A substitution cipher (like the Polybios cipher) replaces each plaintext letter with another according to a system or key. It is broken using frequency analysis, which exploits the fact that certain letters and pairs appear more often than others in a language. $ What is a Shift Cipher and how does it work?\A shift cipher associates every English letter with a number (a=0 to z=25) and shifts every plaintext letter by a secret key K (a number from 0 to 25) using wraparound logic. $ What are the encryption and decryption formulas for a Shift Cipher?\Encryption is defined as C = (M + K) mod 26 and decryption as M = (C - K) mod 26, where C is the ciphertext, M is the plaintext, and K is the secret shift key. $ Why is the Shift Cipher considered insecure?\It is insecure because it has a very small keyspace of only 26 possible keys. An attacker can trivially brute-force the cipher by trying every possible key until the generated plaintext "makes sense." $ What is the Vigenère cipher and how does it differ from a shift cipher?\The Vigenère cipher is a polyalphabetic substitution method that uses a string (keyword) as the key instead of a single letter, shifting each plaintext character by the amount dictated by the corresponding character in the key. $ What is "keyspace" and how does it relate to Vigenère security?\Keyspace is the total number of possible keys; for Vigenère, it is 26 to the power of n (where n is the key length). While this makes brute-force expensive, it is still insecure if the key is short and the ciphertext is long. $ How is a Vigenère key broken without brute-forcing the entire keyspace?\By extracting sub-sequences of characters encrypted by the same key character (every nth character). If the guessed key length is correct, the letter frequency distribution of those sub-sequences will match standard English. $ Explain the encryption and decryption algorithm of a One Time Pad (OTP)?\Encryption is performed by applying the XOR (Exclusive OR) operation between the plaintext message and a random key of the exact same length. Decryption is performed by XORing the ciphertext with that same random key. $ Why is OTP considered to be "perfectly secure"?\It has "perfect secrecy" because the ciphertext provides no information about the plaintext without the key; every possible plaintext of that length is equally likely, regardless of the attacker's computing power. $ Why is it insecure to use the same key to encrypt multiple messages using OTP?\Reusing a key allows an attacker to XOR two ciphertexts together (C1 XOR C2), which cancels out the key and leaves the XOR of the two plaintexts (m1 XOR m2), revealing information about the original messages. $ Why can OTP NOT guarantee integrity?\OTP is a "malleable" cipher; an attacker can flip specific bits in the ciphertext, and those changes will predictably alter the decrypted plaintext without the recipient knowing it was tampered with. $ What is a stream cipher and how does it address OTP limitations?\A stream cipher uses a short secret key as a "seed" for a Pseudo Random Generator (PRG) to create a long pseudo-random key stream (k') that is XORed with the plaintext, avoiding the need for a key as long as the message. $ What are the block and key sizes for the Data Encryption Standard (DES)?\DES operates on data blocks of 64 bits using a 56-bit secret key. $ How does the DES round function operate?\It applies a 48-bit round key to the rightmost 32 bits of a block to produce a 32-bit output. The rightmost 32 bits are then swapped with the leftmost 32 bits, and this is repeated for 16 rounds. $ What is Electronic Code Book (ECB) mode and what is its primary weakness?\ECB is a block cipher mode where each plaintext block is encrypted independently with the same key. Its primary weakness is that identical plaintext blocks produce identical ciphertext blocks, disclosing patterns (like shapes in an image). $ What is Cipher Block Chaining (CBC) mode?\CBC is a mode that uses a random Initialization Vector (IV) and "chains" blocks together so that each ciphertext block depends on all previous plaintext blocks, ensuring identical plaintext blocks result in different ciphertexts. $ What is the difference between a symmetric and an asymmetric cryptographic system?\In a symmetric system, both parties use identical keys for encryption and decryption. In an asymmetric (public key) system, each user has a pair: a public key for encryption and a private key for decryption. $ How do you ensure confidentiality when sending a message using asymmetric cryptography?\You must encrypt the message using the recipient’s public key, which ensures that only the recipient can decrypt it using their corresponding private key. $ What are MAC and HMAC used for?\A Message Authentication Code (MAC), often implemented as a Hash-based MAC (HMAC), is used to ensure both integrity and authenticity by generating a tag with a shared secret key. $ What is Kerckhoffs' Principle?\The principle that a cryptographic system should be secure even if everything about the algorithm is public knowledge, provided that the secret key is chosen at random and kept hidden. $ What is the difference between QKD and PQC in quantum technology?\Quantum Key Distribution (QKD) is a hardware-based method using photons to share keys. Post-Quantum Cryptography (PQC) consists of mathematical algorithms (like ML-KEM) designed to be secure against both classical and quantum computers.
