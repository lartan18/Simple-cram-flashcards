What is XSS?\XSS stands for Cross-Site Scripting; it is an injection attack where malicious scripts are executed in a victim's browser.$
What is the difference between Reflected and Stored XSS?\**Reflected XSS** executes immediately when a user clicks a malicious link, while **Stored XSS** is saved on the server (e.g., in a database) and executes for every user who visits the page.$
What is CSRF?\CSRF stands for Cross-Site Request Forgery; it is an attack that tricks an authenticated user's browser into sending unauthorized requests to a web application.$
How is CSRF typically mitigated?\The primary defenses include the use of **unique anti-CSRF tokens** for actions and configuring **SameSite cookie** settings.$
What is SSRF?\SSRF stands for Server-Side Request Forgery; it occurs when an attacker tricks a trusted server into making requests to internal or external services on their behalf.$
What is XXE?\XXE stands for **XML External Entities**; it is a vulnerability where an weakly configured XML parser processes malicious inputs containing references to external files.$
What is Clickjacking?\Clickjacking uses invisible layers or frames to trick a user into clicking a malicious element while they believe they are interacting with a legitimate page.$
What is Insecure Deserialization?\It occurs when untrusted serialized data is processed as objects without validation, potentially leading to **remote code execution** or data manipulation.$
What is the purpose of "Salt" in password hashing?\A salt is a random number added to a password before hashing to ensure every user has a unique hash, even if they use the same password, protecting against online dictionary attacks.$
What is "Pepper" and how is it stored?\A pepper is a **secret value** added to a password before hashing that is stored in an encrypted form separate from the password database to protect against offline attacks.$
What is XSS Sanitization?\XSS Sanitization involves stripping or escaping HTML tags and **dangerous characters** (like < and >) to prevent malicious scripts from executing in the user's browser.$
What is a CSRF Token?\A CSRF Token is a unique, hidden field or value generated by the server and included in every form that ensures a request originated from the legitimate website rather than from a malicious site.$
What is Authentication?\Authentication is the process of verifying a user's identity based on **something they know** (like a password), **something they have** (like a phone), or **something they are** (like biometrics).$
What is Password Hashing?\Password hashing is the process of converting a plaintext password into a cryptic, fixed-length string using a **one-way function** (like SHA-256), making it impossible to reverse the process.$
What is Login Throttling?\Login Throttling is a countermeasure that slows down login attempts based on how quickly guesses are made, helping prevent **automated bot attacks** and brute-force password guessing.$
