Compare the goals of Information Gathering from an attacker’s vs. a developer's perspective.\Attackers* map attack points and find vulnerabilities to improve efficiency; Developers/Testers determine test coverage and prioritize what needs fixing.$
What are the two main types of information to collect during the gathering phase?\The two types are **Application Structure** (pages, trust zones) and **Data Flow** (parameters, GET/POST requests, responses).$
List three functions of a web debugging proxy in software security.\A debugging proxy can capture and examine requests/responses, manipulate payloads, and be used to perform attacks.$
Define "Google Dorking" and provide its security implication.\It involves using advanced search operators to find sensitive information that was unintentionally exposed on the public internet.$
Explain the logic of a basic SQL Injection using the string 'OR 1=1'.\The attacker injects code into an input field so the query logic becomes 'something OR 1=1', which is always true, bypassing authentication checks.$
Explain how a "Blind SQL Injection" works.\The attacker extracts database data by asking the database **logical yes/no questions** or using binary search to guess table names and schemas.$
List and explain the most effective countermeasure against SQL Injection.\Prepared Statements (with bind variables)* are best because they separate the SQL query structure from the user-provided data, preventing the data from being executed as code.$
Contrast Whitelisting and Blacklisting as injection countermeasures.\Whitelisting* allows only specific "safe" inputs (often using RegExp), while *Blacklisting* filters out known "bad" characters; whitelisting is generally more secure as blacklisting is easy to bypass.$
Why is Session Management necessary in web applications?\It is necessary because HTTP is stateless, meaning the server cannot naturally link two separate requests to the same user without a session token.$
Describe the steps of a "Session Fixation" attack.\1. The attacker visits a site to get an anonymous token; 2. The attacker **overwrites** the user's token with their own; 3. The user logs in; 4. The attacker’s token is elevated to "logged-in," giving them access.$
What is the difference between Session Cookies and Persistent Cookies?\Session cookies* are temporary and stored in memory until the browser closes; Persistent cookies have an expiration date and remain on the device longer.$
